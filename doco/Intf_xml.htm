Interfaces and XML access


XML types

Most XML types do not map natively into pascal classes. ANy type can be specified to be nil, and pascal types cannot represent this. Other types simply have no equivalent in pascal. When you encounter these being used, you can use one of these types:

TIdSoapInteger
 Integer that can represent nil=true
TIdSoapBoolean
 Boolean that can represent nil=true
TIdSoapDouble
 Double that can represent nil=true
TIdSoapString
 String that can represent nil=true
TIdSoapQName
 Represents a QName type that integrates with the XML namespace handling


Direct XML access

Since most SOAP servers work the same way as IndySoap - primarily an RPC mechanism, the types exchanged are generally fairly simple types that map directly into simple types are easily defined classes. However increasingly SOAP is being used to transfer complicated documents that have schemas which are not easy or not worth representing in a native delphi object model. Often these documents are mixed in with classic RPC type SOAP.

To handle this situation, IndySoap includes a type TIdSoapRawXML which allows applications to directly access parts of the soap message
