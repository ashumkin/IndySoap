Installation Procedure

  1. You need the latest Indy release (9.011 or more recent)
  2. You need the latest DUnit source to run the DUnit tests (from http://dunit.sourceforge.net)
  3. You need to have MSXML 4.1 installed to use the xpMsXml xml provider. You also need MsXml installed to run the Dunit tests (from http://msdn.microsoft.com/xml)
  4. Create a SOAP Directory in your Indy directory. This directory is referred to as <indysoap> hereafter
  5. Copy the IndySoap directory structure into <IndySoap> Directory.
    To actually use IndySoap, you only need the files in the root soap directory, but the other files will be useful from time to time
  6. Add <indysoap> to your delphi library path
  7. Review the compiler defines in <indySoap>\IdSoapDefines.inc - you may want to change these to reflect your system and what features you want 
  8. [optional but recommended] Compile and run the DUnit tests in <indysoap>\dunit\IdSoapTests.dpr
  9. Open the IdSoapXX.dpk file, then press the install button.
  10. Package Hell. You might have problems getting IndySoap compiling and installing. here's some tips:
    1. Make sure that your Indy package is updated before installing indysoap
    2. Let delphi modify the package dependencies as it sees fit
    3. when compiling design time packages, delphi compiles off the source in the dcp file, not the unit itself. So weird compile errors can be reported in defiance of the actual source. If you are really having problems, delete any existing indy dcu's and dcp's and recompile the indy package from scratch

If you have problems installing IndySoap, please compile and run <indysoap>\dunit\IdSoapTests.dpr before reporting the problems to the IndySoap team
