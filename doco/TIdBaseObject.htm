TIdBaseObject

 

Throughout IndySoap, every object descends from TIdBaseObject. TIdBaseObject is the basis of object tracking and leak detection, which are used to improve the quality of the code.

Every TIdBaseObject and descendent is entered into a list of live objects, so that the object can check unequivocally that it is valid. This is used throughout IndySoap in precondition checks

You can get the same advantages from TIdBaseObject by inheriting your own objects from it

 

class function GetLiveObjectCount: Cardinal;

Return the number of live objects of this type (this count does not include descendent types)

 

property SerialNumber: Integer

It's possible that an object is freed and a new object is created in the same memory location, so that it appears to be object. For this reason, every object created get's a unique serial number that never changes through the lifetime of the object. You can use this to check that the objet hasn't changed (you must store this outside the object cause if a new object is created, this will be changed)

 

procedure AskForBreakPointOnFree;

If this is called, then when the object is freed, a debugger breakpoint will be called. This will be called even if the object is erroneously freed as another object (but not if it is erroneously freed using freemem or similiar)

 

function TestValid(AClassType: TClass = NIL): Boolean;

Checks that the object is in the list of current live objects. If it is, checks that the class is of type AClassType (or a descendent thereof)
